About the Project

Full-Stack Embodied AI

A full-stack embodied AI system enabling a quadruped robot to interpret natural language tasks autonomously, explore an environment, and retrieve target objects — bridging the gap between high-level human intent and real-world robotic execution.

Architecture

GoldenGait integrates a vision-language model frontend with a hierarchical motion planning backend, enabling seamless closed-loop reasoning from task description and raw sensor input to low-level joint control.

Frequently Asked Questions

Q & A

Can GoldenGait operate fully autonomously, without a human in the loop? ‹
Yes — once given an initial task in natural language, GoldenGait operates fully autonomously. It handles exploration, object localization, and navigation end-to-end.
How does the system handle novel objects it has never seen before? ‹
The VLM backbone provides strong zero-shot generalization. By leveraging a large pre-trained vision-language model, GoldenGait can identify and reason about objects it was never explicitly trained on — using open-vocabulary detection and contextual reasoning.
How long does the robot take to complete a typical fetch task? ‹
Task completion time varies with environment size and object location. In our DARPA challenge runs, average task completion time was around 5 minutes.
Is the code open source? ‹
Yes! We are releasing the full codebase and a simulation interface on GitHub. Please see the GitHub link at the top of this page. We hope this accelerates research in embodied AI and legged robotics.
How does the agent communicate with the environment? ‹
Our agent communicates with the simulation through ROS topics, including sensor data streams (RGB-D, proprioception, TF) and command topics for high-level actions and low-level joint control.
What are some limitations of GoldenGait? ‹
On a low level, we assume perfect state estimation and rely on a simulation physics during training. Real-world deployment will require additional robustness measures for perception noise and other sim-to-real gaps. On a high level, the agent can struggle with highly ambiguous instructions.
